## Maven Dependency Mediation Rules

- When two declarations resolve the same artifact at the **same depth** in the dependency tree, Maven picks the **first one declared** in `pom.xml` ‚Äî "nearest + first wins".
- `<dependencyManagement>` with a BOM (`<type>pom</type> <scope>import</scope>`) **always wins** over transitive resolution regardless of declaration order or depth. This is the safe pattern for managing a family of related artifacts (e.g. all `io.opentelemetry:*`).
- Never split SDK and exporter versions into separate properties for the same OTel release train ‚Äî they must be co-versioned.

## OTel Version Compatibility

- `GrpcSenderConfig` was introduced in `opentelemetry-sdk-common` at version **1.43.0**.
- Any combination where `opentelemetry-exporter-otlp` ‚â• 1.43.0 but `opentelemetry-sdk-common` < 1.43.0 (resolved transitively) will produce `NoClassDefFoundError: io/opentelemetry/sdk/common/export/GrpcSenderConfig` at JVM startup.
- `opentelemetry-semconv:1.24.0-alpha` is not part of the BOM ‚Äî it must always be declared explicitly with its version. The leading space in `" 1.24.0-alpha"` in pom.xml is harmless but worth tidying.
- OTel artifacts to always keep co-versioned (managed by BOM): `opentelemetry-api`, `opentelemetry-sdk`, `opentelemetry-exporter-otlp`, `opentelemetry-sdk-common` (transitive).

## GitHub Actions ‚Äî Service Containers

- `volumes:` inside `services:` blocks only works on **self-hosted runners** or runners that have Docker available with bind-mount capability. GitHub-hosted `ubuntu-latest` runners support it but the path must be relative to `$GITHUB_WORKSPACE`. The `otel-collector` service uses a volume mount ‚Äî if it ever fails, check this first.
- `--health-cmd` using `wget` requires the container image to have `wget`. Prometheus (`prom/prometheus`) and Pushgateway (`prom/pushgateway`) lightweight images do have it. If the image changes, this may break.
- Service container health checks use `--health-interval`, `--health-timeout`, `--health-retries` passed via `options:`. The workflow then additionally does manual curl-based waits in steps ‚Äî belt-and-braces approach.

## Dependabot Behaviour

- Dependabot always uses branch names matching `dependabot/<ecosystem>/<dependency>-<version>`.
- `auto-merge: true` in `dependabot.yml` only works if the GitHub repo has **"Allow auto-merge"** enabled in Settings ‚Üí General.
- Dependabot re-runs CI when you push additional commits to a Dependabot branch ‚Äî it does not close/reopen the PR.
- When you push to a Dependabot branch manually, Dependabot loses its ability to rebase/recreate the PR automatically (it was modified). Comment `@dependabot recreate` to reset if needed.
- On personal accounts, Dependabot `auto-merge` requires the user to have the "Allow auto-merge" org/repo setting on. On orgs it may need additional policy enablement.

## Pact Contract File

- `src/test/resources/pacts/reqResUserConsumer-reqResUserProvider.json` is auto-generated by the Pact consumer test on each local test run.
- The `pact-jvm.version` field in the JSON reflects the version of `au.com.dius.pact:consumer` in use ‚Äî this will change naturally with dependency bumps and should be committed alongside `pom.xml` changes.

## GitHub Copilot for PRs / Models

- `actions/ai-inference` (backed by GitHub Models) requires **GitHub Copilot Business or Enterprise** on the organisation.
- On personal repos, GitHub Models API is accessible via `GITHUB_TOKEN` with `models: read` permission ‚Äî available in public repos as of early 2026.
- The `@github-copilot try to fix` PR comment trigger requires "Copilot for Pull Requests" to be enabled per-repo in Settings ‚Üí Copilot.

## Git Workflow Notes

- Local `otel_observability` branch has diverged significantly from `origin/otel_observability` (12 local, 24 remote). Do not push local `otel_observability` without rebasing ‚Äî it will overwrite remote work.
- Always run `git branch` before any commit/push to confirm the active branch.
- Always stash or pop before switching branches if `pom.xml` has uncommitted changes.
- The stash `fix: add opentelemetry-bom` may still exist in the stash list ‚Äî run `git stash list` to check and drop if no longer needed.
